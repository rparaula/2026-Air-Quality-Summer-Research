You were right to question it. The pollutant parameter codes were fine, but the script was reading the wrong response field.

EPA AQS is returning monitor rows under top-level Data, not Body. The script was doing payload.get("Body", []), so every successful API response looked empty. I patched collect_AQS_monitors.py locally to read Data first, then fall back to Body.

I verified with a smoke test:

.\.venv\Scripts\python.exe .\collect_AQS_monitors.py `
  --email test@aqs.api `
  --key test `
  --counties Harris `
  --pollutants ozone `
  --start-date 2026-01-01 `
  --end-date 2026-05-22 `
  --active-as-of 2026-05-22 `
  --request-delay 0 `
  --output-file "AQS_data\monitor_locations\_debug_harris_ozone.csv"
That returned 17 Harris ozone monitor rows. I also added row-count logging and a fail-fast guard so future no-row runs won’t silently write a header-only CSV unless you pass --allow-empty.

Worked for 4m 29s
Built AQS_concentrations.py.

It reads the latest collect_AQS_monitors.py output by default, queries AQS sampleData/bySite, and writes a complete hourly monitor-site grid to AQS_data\air_data. Pollutants a station cannot measure, or hours with no AQS reading, stay blank/NaN. If AQS returns multiple readings for the same site/pollutant/hour, the script averages them into one value.
